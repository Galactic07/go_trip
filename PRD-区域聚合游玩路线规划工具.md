## 1. 产品概述

### 1.1 产品背景

年轻旅行者日常在社交媒体（小红书、抖音等）高频"种草"各种景点、餐厅、打卡地，但这些碎片化的地点信息散落在相册截图、收藏夹、备忘录中，难以统一管理和规划。当收集地点足够多时，用户往往无法直观地发现哪些地点地理上相近、可以安排在同一天游玩。

跨区域游玩存在明显的规划痛点：一个区域内零散分布的多个心愿地点，由于缺乏聚合视角，常常被拆分到不同天次中，导致单日路线绕路、时间浪费、体验打折。

### 1.2 产品定位

一款基于中国地图的**心愿地点收集与区域聚合规划工具（Web应用）**。用户在地图上持续添加想去的地方，系统自动将地理上相近的地点按行政区划进行聚类归组，帮助用户直观发现"哪些地点可以安排在同一天"，从而高效规划单日游玩路线。

### 1.3 核心价值主张

| 维度 | 价值 |
|------|------|
| **收集极轻** | 随手输入地名、点击地图POI，或**粘贴分享链接自动解析**，3秒完成添加 |
| **聚合惊喜** | 地图气泡自动呈现"这片区域居然有这么多想去的地方" |
| **探索发现** | 直接在地图上点击任意位置，发现周边真实POI并加入心愿单 |
| **规划可控** | 基于区级聚类分组，智能排序生成可控的单日路线，含**时间预算与费用估算** |
| **状态管理** | 地点状态追踪（想去/已去过/待定/不想去），心愿单不再混乱 |
| **天气感知** | 路线规划时结合天气预报和室内外标签，给出出行建议 |
| **数据持久化** | MySQL数据库存储，数据安全可靠 |

### 1.4 与竞品差异化

| 对比维度 | 高德/百度地图 | 携程/马蜂窝 | **本产品** |
|---------|-------------|------------|-----------|
| 核心场景 | 导航出行 | 旅游攻略浏览 | **心愿收集→区域聚合→路线规划** |
| 地点管理 | 无收藏聚合 | 攻程模板固定 | **用户自定义心愿单+自动聚类** |
| 聚合维度 | 无 | 按城市/主题 | **市/区/街道三级地理聚类** |
| 规划方式 | 手动规划单次行程 | 跟随预设攻略 | **基于聚合分组智能推荐路线顺序** |
| 探索模式 | 无 | 无 | **直接点击地图发现POI** |
| 适用人群 | 所有人 | 有明确目的地的人 | **碎片化种草积累的年轻旅行者** |

## 3. 目标用户

### 3.1 核心用户画像

**小林，25岁，深圳打工人，热爱探索城市**

- 日常在小红书/抖音高频刷到各种"宝藏店铺""绝美打卡地"
- 手机相册里有大量截图，但从未系统整理过想去的地方
- 周末出游经常到了某地才发现，另一个想去的地方就在附近
- 最大痛点：出发前不知道哪些地方挨着、应该怎么排序路线
- 对"好看、有成就感"的产品界面有天然好感
- 技术接受度高，愿意尝试新工具

### 3.2 用户行为特征

```
收集阶段（高频、碎片化）              规划阶段（低频、集中）
─────────────────────               ─────────────────────
· 刷到种草内容 → 搜索框输入添加       · 打开地图查看分布
· 点击地图上的POI → 一键加入          · 发现某区域聚合了大量地点
· 朋友推荐某个店 → 记下来             · 进入分组查看详情
· 持续积累数天/数周                   · 区级分组生成路线
                                    · 执行路线并标记进度
```

**关键洞察**：收集和规划是完全不同的心理状态。收集时追求"快"（不中断当前浏览），规划时追求"清晰"（一眼看出哪些地方在一起）。

### 3.3 用户目标与痛点

| 用户目标 | 当前痛点 | 产品解决方案 |
|---------|---------|------------|
| 把想去的地方记下来，不遗忘 | 散落在相册/收藏/备忘录中，难以查找 | 统一地图标记 + 数据库持久化，所有心愿一目了然；**粘贴链接自动解析（F-13）** |
| 知道哪些地方可以一起玩 | 地点碎片化，无法感知地理邻近关系 | 市/区/街道三级自动聚类，气泡数量直观呈现 |
| 规划合理的单日路线 | 手动查地图距离，费时且容易遗漏 | 基于区级聚类分组智能推荐路线顺序；**时间预算与费用估算（F-12）** |
| 发现新的好玩地方 | 需要主动搜索才知道有什么 | 直接在地图上点击，自动获取周边真实POI |
| 灵活调整计划 | 计划变更需要重新搜索查地图 | 拖拽排序、一键增删，实时更新 |
| 心愿单不混乱 | 地点越来越多，分不清哪些去过、哪些不想去 | **地点状态管理（F-11）：想去/已去过/待定/不想去** |
| 出行不被天气影响 | 兴冲冲出发才发现下雨或太热 | **天气感知与室内外标签（F-14）：结合天气预报给出出行建议** |
| 快速添加种草地点 | 复制链接→打开APP→记住名字→回来搜索，四步太长 | **分享链接自动解析（F-13）：粘贴即识别，一步完成** |

```

2. 输入过程中实时显示联想下拉列表（debounce 300ms）

5. 地图自动平移到该位置并标记
6. 搜索框清空，回归待搜索状态
7. 数据通过API保存到MySQL数据库
8. 该地点所属区域的聚合气泡数字 +1（若已有其他标记）
```

### 4.15 地图标记重叠优化 ✅ v2.1 已实现

| 属性 | 描述 |
|------|------|
| **功能描述** | 当两个或以上地点坐标距离 < 30米时，标记自动呈圆形散开排列，避免完全重叠无法点击 |
| **用户价值** | 解决同一建筑/相邻建筑的多个地点在地图上叠在一起的问题 |
| **优先级** | T0（体验修复） |
| **实现状态** | ✅ v2.1 完成 |
| **核心文件** | `MapMarkers.tsx` |

#### 散开算法

- **检测阈值**: 两点直线距离 < ~30 米（经纬度差 < 0.0003）
- **偏移半径**: 约 25 米（0.00025）
- **排列方式**: 重叠标记围绕几何中心呈均匀圆形分布
- **效果**: 相近地点的标记不再完全叠在一起，每个都能独立点击
