## 1. 需求背景

### 1.1 用户痛点

路线规划完成后，用户面临**"做好了怎么发出去"**的问题：

| 场景 | 痛点 |
|------|------|
| 发给朋友一起参考 | 只能截图，长路线截不全、文字模糊 |
| 发朋友圈/小红书分享 | 需要手动拼图、打码、排版，费时费力 |
| 保存到相册留底 | 没有导出功能，只能靠浏览器书签回来找 |
| 打印出来随身带 | 完全不支持 |

### 1.2 用户价值

路线导出是**单用户阶段最高频的分享需求**，也是产品从"自用工具"走向"社交传播"的第一块跳板。一张精美的路线卡片图，本身就是最好的产品宣传物料。

### 3.2 方案一：图片导出（P0）

#### 3.2.1 技术选型对比

| 方案 | 技术栈 | 优点 | 缺点 | 推荐度 |
|------|--------|------|------|--------|
| **A. html2canvas** | `html2canvas` 库 | 直接截图 DOM，样式完全一致，开发成本极低 | 字体渲染偶发模糊，CSS 滤镜/阴影可能失效 | ⭐⭐⭐⭐⭐ |
| B. dom-to-image | `dom-to-image-more` | 矢量 SVG 输出，放大不失真 | 对现代 CSS 支持不如 html2canvas | ⭐⭐⭐ |
| C. Canvas 手绘 | 原生 Canvas API | 完全可控，性能最好 | 开发量巨大，样式调整困难 | ⭐⭐ |
| D. 后端渲染 | Go + headless Chrome | 服务端生成，客户端无依赖 | 增加后端复杂度，需要 Puppeteer | ⭐⭐ |

**结论：推荐使用 html2canvas（方案 A）**，理由：
1. 前端纯客户端完成，不增加后端负担
2. 复用现有 RoutePlanner 的 DOM 结构，开发量最小
3. 社区成熟，问题有现成解决方案
4. 1.4MB 的 bundle 大小可接受（懒加载）

**Step 3：导出核心逻辑 `exportImage.ts`**

```typescript
// lib/exportImage.ts
import html2canvas from 'html2canvas';

export interface ExportImageOptions {
  format?: 'png' | 'jpeg';
  quality?: number;       // jpeg 质量 0-1
  scale?: number;         // 缩放倍数，2 = 2倍分辨率
  filename?: string;      // 下载文件名
}

export async function exportCardAsImage(
  element: HTMLElement,
  options: ExportImageOptions = {}
): Promise<void> {
  const {
    format = 'png',
    quality = 0.92,
    scale = 2,  // 2倍渲染，保证高清
    filename = `路线导出_${Date.now()}`,
  } = options;

  try {
    const canvas = await html2canvas(element, {
      scale,                         // 高清输出
      useCORS: true,                // 允许跨域图片
      allowTaint: false,            // 不允许污染（安全）
      backgroundColor: '#ffffff',   // 白色背景
      logging: false,               // 生产环境关闭日志
      // 关键：确保字体已加载
      onclone: (clonedDoc) => {
        // 在克隆的 DOM 中注入字体预加载
        const style = clonedDoc.createElement('style');
        style.textContent = `
          @import url('https://fonts.googleapis.com/css2?family=Noto+Sans+SC:wght@400;700&display=swap');
          * { font-family: 'Noto Sans SC', 'PingFang SC', sans-serif !important; }
        `;
        clonedDoc.head.appendChild(style);
      },
    });

    // 转为 Blob 并下载
    return new Promise((resolve) => {
      canvas.toBlob(
        (blob) => {
          if (!blob) { throw new Error('导出失败：无法生成图片'); }
          const url = URL.createObjectURL(blob);
          const a = document.createElement('a');
          a.href = url;
          a.download = `${filename}.${format}`;
          document.body.appendChild(a);
          a.click();
          document.body.removeChild(a);
          URL.revokeObjectURL(url);
          resolve();
        },
        `image/${format}`,
        quality
      );
    });
  } catch (err) {
    console.error('图片导出失败:', err);
    throw new Error('图片导出失败，请重试');
  }
}
```

#### 3.2.4 关键问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| **中文字体模糊/缺失** | html2canvas 默认不加载 Web 字体 | `onclone` 钩子中注入 Google Fonts CDN 的 Noto Sans SC |
| **图片跨域报错** | 地图截图/第三方图片触发 CORS | 设置 `useCORS: true`，服务端配置 `Access-Control-Allow-Origin` |
| **导出内容截断** | 长路线超出视口 | 导出模板用 `position: absolute; left: -9999px` 脱离文档流，不限制高度 |
| **高清屏模糊** | 默认 1x 渲染 | `scale: 2` 或 `devicePixelRatio` |
| **CSS 变量/滤镜失效** | html2canvas 部分 CSS 不支持 | 导出模板使用内联样式，避免 CSS 变量和 `filter` |
| **Tailwind 类名不生效** | html2canvas 不执行 JS/CSS 构建 | 导出模板改用**内联样式或独立 CSS 文件**，不依赖 Tailwind CDN |

#### 3.3.3 实现代码

```typescript
// lib/exportText.ts
export interface RouteExportData {
  title: string;
  places: Array<{
    name: string;
    address: string;
    stayDuration: number;
    estimatedCost?: number;
    tags: string[];
    note?: string;
    sceneType?: string;
  }>;
  segments: Array<{
    transport: 'walk' | 'drive';
    duration: string;
    distance: string;
  }>;
  summary: {
    totalPlaces: number;
    totalPlayTime: string;
    totalTransitTime: string;
    totalCost?: number;
  };
  weather?: {
    temperature: number;
    condition: string;
  };
}

export function generateRouteText(data: RouteExportData): string {
  const lines: string[] = [];

  // 标题
  lines.push(`🗺️ ${data.title}`);
  lines.push('━━━━━━━━━━━━━━━━━━━━');
  lines.push('');

  // 地点列表
  data.places.forEach((place, i) => {
    lines.push(`① ${place.name}`);
    lines.push(`📍 ${place.address}`);
    lines.push(`🕐 建议停留 ${place.stayDuration}min${place.estimatedCost ? `  ·  💰 约 ¥${place.estimatedCost}` : ''}`);
    if (place.tags.length > 0) {
      lines.push(`🏷️ ${place.tags.join(' · ')}`);
    }
    if (place.note) {
      lines.push(`📝 ${place.note}`);
    }

    // 路段信息
    if (i < data.segments.length) {
      const seg = data.segments[i];
      const icon = seg.transport === 'walk' ? '🚶' : '🚗';
      lines.push(`  ↓ ${icon} ${seg.transport === 'walk' ? '步行' : '驾车'}${seg.duration} · ${seg.distance}`);
    }
    lines.push('');
  });

  // 分隔线
  lines.push('━━━━━━━━━━━━━━━━━━━━');

  // 概览
  lines.push('📊 路线概览');
  lines.push(`📍 ${data.summary.totalPlaces}个地点  ·  🕐 游玩${data.summary.totalPlayTime}  ·  🚶 交通${data.summary.totalTransitTime}`);
  if (data.summary.totalCost) {
    lines.push(`💰 约 ¥${data.summary.totalCost}`);
  }
  lines.push(`⏰ 预计总耗时：${data.summary.totalPlayTime} + ${data.summary.totalTransitTime}`);

  // 天气
  if (data.weather) {
    lines.push('');
    lines.push(`🌤 天气：${data.weather.temperature}°C · ${data.weather.condition}`);
  }

  lines.push('━━━━━━━━━━━━━━━━━━━━');
  lines.push('📱 由「区域聚合游玩路线规划工具」生成');

  return lines.join('\n');
}

// 复制到剪贴板
export async function copyRouteAsText(data: RouteExportData): Promise<void> {
  const text = generateRouteText(data);
  try {
    await navigator.clipboard.writeText(text);
  } catch (err) {
    // 降级方案：使用 textarea 选中复制
    const textarea = document.createElement('textarea');
    textarea.value = text;
    textarea.style.position = 'fixed';
    textarea.style.left = '-9999px';
    document.body.appendChild(textarea);
    textarea.select();
    document.execCommand('copy');
    document.body.removeChild(textarea);
  }
}

// 下载为文本文件
export function downloadRouteAsText(data: RouteExportData, filename?: string): void {
  const text = generateRouteText(data);
  const blob = new Blob([text], { type: 'text/plain;charset=utf-8' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = filename || `路线_${data.title}_${Date.now()}.txt`;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
  URL.revokeObjectURL(url);
}
```

### 3.4 方案三：PDF 导出（P1）

#### 3.4.1 技术选型

| 方案 | 技术栈 | 优点 | 缺点 |
|------|--------|------|------|
| **A. jsPDF + html2canvas** | `jspdf` + `html2canvas` | 成熟方案，控制力强 | 中文字体需要手动嵌入 |
| B. Puppeteer 后端渲染 | Go + headless Chrome | 完美还原，支持分页 | 后端新增依赖，运维成本 |
| C. React-PDF | `@react-pdf/renderer` | 声明式 PDF 组件 | 学习成本，样式系统独立 |

**结论：推荐方案 A（jsPDF + html2canvas）**，理由：纯前端完成、与图片导出复用 html2canvas 逻辑、开发量最小。

#### 3.4.2 实现思路

```typescript
// lib/exportPdf.ts
import jsPDF from 'jspdf';
import html2canvas from 'html2canvas';

export async function exportCardAsPdf(
  element: HTMLElement,
  options: { filename?: string; title?: string } = {}
): Promise<void> {
  const { filename = `路线导出_${Date.now()}`, title = '路线导出' } = options;

  // Step 1: 用 html2canvas 渲染为 Canvas
  const canvas = await html2canvas(element, {
    scale: 2,
    useCORS: true,
    backgroundColor: '#ffffff',
  });

  // Step 2: 创建 PDF
  const pdf = new jsPDF({
    orientation: 'portrait',
    unit: 'mm',
    format: 'A4',
  });

  // Step 3: 计算图片在 PDF 中的尺寸（等比缩放适应 A4 宽度）
  const imgWidth = 190; // A4 可用宽度约 190mm
  const imgHeight = (canvas.height * imgWidth) / canvas.width;

  // Step 4: 如果内容超过一页，需要分页
  const pageHeight = 277; // A4 高度约 297mm，留 20mm 边距
  let heightLeft = imgHeight;
  let position = 10; // 顶部边距 10mm

  const imgData = canvas.toDataURL('image/png');

  // 第一页
  pdf.addImage(imgData, 'PNG', 10, position, imgWidth, imgHeight);
  heightLeft -= (pageHeight - 20);

  // 后续页
  while (heightLeft > 0) {
    position = -(pageHeight - 20 - heightLeft - imgHeight) + 10;
    pdf.addPage();
    pdf.addImage(imgData, 'PNG', 10, position, imgWidth, imgHeight);
    heightLeft -= (pageHeight - 20);
  }

  // Step 5: 下载
  pdf.save(`${filename}.pdf`);
}
```

#### 3.5.2 KML 生成示例

```typescript
// lib/exportKml.ts
export function generateKML(places: Place[], routeTitle: string): string {
  const kmlHeader = `<?xml version="1.0" encoding="UTF-8"?>
<kml xmlns="http://www.opengis.net/kml/2.2">
  <Document>
    <name>${escapeXml(routeTitle)}</name>
    <description>由「区域聚合游玩路线规划工具」生成</description>`;

  const kmlPlacemarks = places
    .map(
      (place, i) => `
    <Placemark>
      <name>${i + 1}. ${escapeXml(place.name)}</name>
      <description><![CDATA[
        ${place.address ? `📍 ${place.address}<br/>` : ''}
        ${place.note ? `📝 ${place.note}<br/>` : ''}
        ${place.tags?.length ? `🏷️ ${place.tags.join(', ')}` : ''}
      ]]></description>
      <Point>
        <coordinates>${place.lng},${place.lat},0</coordinates>
      </Point>
    </Placemark>`
    )
    .join('');

  const kmlFooter = `
  </Document>
</kml>`;

  return kmlHeader + kmlPlacemarks + kmlFooter;
}

function escapeXml(str: string): string {
  return str
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;')
    .replace(/"/g, '&quot;')
    .replace(/'/g, '&apos;');
}

export function downloadKML(places: Place[], routeTitle: string): void {
  const kml = generateKML(places, routeTitle);
  const blob = new Blob([kml], { type: 'application/vnd.google-earth.kml+xml' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `${routeTitle}.kml`;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);
  URL.revokeObjectURL(url);
}
```

## 7. 风险与注意事项

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| html2canvas 中文字体渲染 | 字体缺失导致方块或模糊 | 预加载 Google Fonts Noto Sans SC，提供降级方案 |
| 长路线导出内容截断 | 超过一屏的路线无法完整截图 | 脱离视口渲染（`position: absolute; left: -9999px`），不限制高度 |
| 浏览器兼容性（Clipboard API） | 旧版浏览器不支持 | 降级为 `document.execCommand('copy')` |
| 导出图片体积过大 | 2x 高清 + 长图可能导致文件 > 5MB | 提供 `scale` 选项，jpeg 格式可选 |
| 地图截图不显示 | 地图 Canvas 被 CORS 拦截 | 地图区域不截图，导出模板中省略地图部分 |
| 用户未生成路线就点击导出 | 无数据可导出 | 按钮置灰 + Tooltip 提示"请先规划路线" |
